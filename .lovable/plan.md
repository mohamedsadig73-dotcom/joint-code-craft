
## الهدف

ترقية شاشة "سجل الاستلام" (`/boxes` → تبويب الاستلامات) إلى مستوى نظام مخازن احترافي عبر:
1. نظام فلاتر شامل ومنظم (شريط جانبي + شرائح فلاتر نشطة + حفظ مجموعات).
2. طباعة منظمة للفواتير لكل شركة/مورد، مع ملخص لكل فاتورة وإمكانية طباعة فاتورة واحدة أو كل فواتير شركة معينة.

## المرحلة 1 — نظام الفلاتر الاحترافي

### أ. توسيع الفلاتر المتاحة
الموجود حالياً: بحث نصي، الوجهة، الحالة، نوع التغليف فقط.
المطلوب إضافته:
- **الشركة (المورد)** — قائمة منسدلة قابلة للبحث (Combobox) من الموردين الموجودين.
- **رقم الفاتورة** — قائمة منسدلة قابلة للبحث.
- **رقم الصندوق** — قائمة منسدلة قابلة للبحث.
- **نطاق التاريخ** — من/إلى مع اختصارات (اليوم، 7 أيام، الشهر، السنة).
- **نطاق الكمية** — حد أدنى/أقصى.
- **مع صورة / بدون صورة** — تبديل ثلاثي.
- **منشئ السطر** — للإدارة فقط.

### ب. واجهة الفلاتر الجديدة
- **زر "فلاتر" واحد** في شريط الأدوات يفتح **لوحة جانبية (Sheet)** على اليمين بكل الفلاتر مرتبة في أقسام مطوية:
  - تصنيف الشحنة (الوجهة، الحالة، التغليف)
  - الشركة والفاتورة (المورد، رقم الفاتورة، رقم الصندوق)
  - التاريخ والكمية
  - أخرى (الصورة، المنشئ)
- **شريط شرائح فلاتر نشطة** أسفل البحث — كل فلتر مفعّل يظهر كـ Chip مع زر `×` لإزالته فردياً، وزر "مسح الكل".
- **عداد فلاتر نشطة** على زر "فلاتر".
- **حفظ مجموعات فلاتر** (Presets) في `localStorage` — مثل: "شحنات المغرب الشهر الحالي"، "صناديق غير مشحونة لمورد X".

### ج. الترتيب البصري
على الديسكتوب يصبح شريط الأدوات بهذا الترتيب من اليمين لليسار (RTL):
```text
[+ إضافة استلام] [⋯ المزيد]   …   [طباعة] [فلاتر (3)] [🔍 بحث]
```
وتحت شريط الأدوات مباشرة: شريط شرائح الفلاتر النشطة.

## المرحلة 2 — طباعة الفواتير لكل شركة

### أ. تجميع تلقائي للفواتير
حساب من الـ receipts المعروضة (بعد الفلترة):
- مجموعة لكل (شركة + رقم فاتورة) — كل مجموعة = "فاتورة كاملة".
- إذا لم يكن هناك رقم فاتورة، تُجمّع تحت "بدون رقم فاتورة" لكل مورد.

### ب. حوار "طباعة فواتير الشركات"
زر جديد بجانب زر الطباعة الحالي يفتح حواراً يحتوي:
- **شجرة جانبية**: قائمة الشركات → تحت كل شركة قائمة فواتيرها مع عدد البنود ومجموع الكميات.
- **مربعات اختيار**: يمكن اختيار شركة كاملة (كل فواتيرها) أو فاتورة بعينها أو عدة فواتير من شركات مختلفة.
- **خيارات تنسيق**:
  - فاتورة واحدة لكل صفحة (افتراضي).
  - دمج كل الفواتير في مستند واحد مع فواصل صفحات.
  - تضمين الصور أو إخفاءها.
  - عرض المجاميع لكل فاتورة + مجموع كلي للشركة.
- زر **"طباعة"** يولّد HTML منسق ويفتحه عبر نمط `iframe + Blob` المعتمد في النظام.

### ج. تصميم ورقة الفاتورة (A4 عمودي)
لكل فاتورة:
```text
┌────────────────────────────────────────────────────┐
│  [شعار]   نظام إدارة المخازن — استلام بضاعة         │
│           فاتورة رقم: INV-2025-0123                 │
│           التاريخ: 10/05/2026                        │
├────────────────────────────────────────────────────┤
│  المورد: شركة إبراهيم محمد الخليفي التجارية         │
│  الوجهة: المغرب          نوع التغليف: مختلط         │
├────────────────────────────────────────────────────┤
│  # │ صورة │ رقم القطعة │ الوصف │ الكمية │ الوحدة │ │
│  1 │ ...  │ TCP02      │ ...   │ 12     │ PCS    │ │
│  ... جدول البنود ...                                │
├────────────────────────────────────────────────────┤
│  إجمالي البنود: 24    إجمالي الكميات: 348           │
│  عدد الصناديق: 5      الحالة الغالبة: مستلم         │
├────────────────────────────────────────────────────┤
│  توقيع المستلم: __________   توقيع المورد: ________ │
│  ختم المخزن              صفحة 1 من 3                │
└────────────────────────────────────────────────────┘
```
- خط Segoe UI Arabic، تواريخ DD/MM/YYYY، أرقام غربية، عملة "ريال" حيث ينطبق.
- ألوان الهوية: Navy (#1A1F2C) للترويسة، Gold (#FFD700) لخطوط الفصل.
- في النسخة المدمجة لشركة كاملة: صفحة غلاف بمجاميع الشركة عبر كل فواتيرها قبل الفواتير.

### د. ربط الفلتر بالطباعة
زر سريع في شريط شرائح الفلاتر: عند تفعيل فلتر "المورد"، يظهر زر **"طباعة كل فواتير {اسم الشركة}"** يطبع مباشرة دون فتح الحوار.

## التفاصيل التقنية

- **ملفات للإنشاء**:
  - `src/components/boxes/filters/ReceiptsFiltersPanel.tsx` (لوحة الفلاتر الجانبية).
  - `src/components/boxes/filters/ActiveFiltersBar.tsx` (شرائح الفلاتر النشطة).
  - `src/components/boxes/filters/FilterPresetsMenu.tsx` (حفظ/تحميل المجموعات).
  - `src/components/boxes/print/SupplierInvoicesPrintDialog.tsx` (حوار اختيار الفواتير).
  - `src/components/boxes/print/buildSupplierInvoiceHTML.ts` (مولّد HTML).
  - `src/hooks/useReceiptsFilters.ts` (state + منطق الفلاتر + حساب الشرائح).
- **ملفات للتعديل**:
  - `src/components/boxes/ReceiptsTab.tsx` — استبدال state/JSX الفلاتر الحالي بالنظام الجديد، إضافة زر طباعة الفواتير.
  - `src/locales/ar.ts` و `src/locales/en.ts` — مفاتيح ترجمة جديدة (filters, supplier, invoice, presets, printSupplierInvoices, perInvoice, mergedDocument, ...).
- **التقيد بمعايير المشروع**:
  - مكونات < 300 سطر، استخراج المنطق إلى hook.
  - RTL عبر CSS منطقي (start/end).
  - بدون hardcoded strings.
  - الطباعة بأسلوب `iframe + Blob` في الويب و `webContents.printToPDF` في Electron.
  - استخدام مكونات shadcn الموجودة (Sheet, Combobox عبر Command, Checkbox, Collapsible).
- **الأداء**: حساب التجميع والمجاميع داخل `useMemo`، وتحميل صور الطباعة كسولاً.

## ما لن يتغير في هذا الإصدار
- لا تغييرات في قاعدة البيانات (كل البيانات اللازمة موجودة في `box_receipts`).
- لا تغيير على شاشات أخرى غير سجل الاستلام.
- زر "طباعة بالعرض الحالي" الموجود يبقى كما هو إلى جانب الزر الجديد.

سأنفذ المراحل بالترتيب وأرفع نسخة Desktop جديدة (Hot-Swap) كما تنص سياسة النشر.
